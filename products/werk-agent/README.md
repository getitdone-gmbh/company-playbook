## Welches Problem wir lösen

Unsere Kunden besitzen Maschinendaten, Prozesswissen, Qualitätsdaten, Störmeldungen, Excel-Auswertungen sowie Historian- oder MES-Daten. Ihnen fehlen aber häufig genügend Data Scientists, um daraus schnell verwertbare Erkenntnisse zu gewinnen.

Für uns ist das nicht ausschließlich ein Datenproblem.

> **Es ist ein Übersetzungsproblem zwischen Produktionswissen, Datenanalyse und Umsetzung.**

## Warum der Fit stark ist

Der Product-Persona-Fit ist bei uns besonders stark, weil wir WERK Agent nicht als allgemeinen KI-Chatbot vermarkten, sondern als Werkzeug für konkrete Produktionsfragen:

- Warum steigt der Ausschuss?
- Welche Parameter beeinflussen die Qualität?
- Was geschieht vor einem Stillstand?
- Warum unterscheiden sich zwei Schichten?
- Welche Prozesswerte korrelieren mit einer Störung?
- Welche Maßnahmen haben tatsächlich geholfen?

## Wie wir den Ablauf verändern

**Problem:** Eine Produktionslinie erzeugt wiederholt Qualitätsabweichungen. Die Daten liegen vor, aber verteilt auf Maschinenexporte, Laborwerte und Excel-Dateien.

| Heute (ohne uns) | Mit WERK Agent |
|---|---|
| 1. Daten werden manuell exportiert. | 1. Der Ingenieur beschreibt das Qualitätsproblem. |
| 2. Ein Analyst / Data Scientist muss gefunden werden. | 2. Vorhandene Daten werden angebunden oder hochgeladen. |
| 3. Produktionswissen wird in mehreren Gesprächen erklärt. | 3. Der Agent untersucht Zusammenhänge und Auffälligkeiten. |
| 4. Ein Notebook / eine einmalige Auswertung entsteht. | 4. Ergebnisse werden fachlich verständlich dargestellt. |
| 5. Die Erkenntnis erreicht die Mitarbeiter verspätet. | 5. Hypothesen werden durch den Produktionsmitarbeiter bewertet. |
| 6. Die Analyse wird beim nächsten Problem erneut aufgebaut. | 6. Maßnahmen werden dokumentiert. |
| | 7. Nach der Umsetzung wird die Wirkung überprüft. |
| | 8. Erfolgreiche Analysen werden als wiederverwendbare Vorgehensweise gespeichert. |

## Wie wir uns differenzieren

Wir konkurrieren nicht nur mit anderen Industrial-AI-Produkten, sondern auch mit Excel, manuellen Auswertungen, BI-Tools, Data-Science-Projekten, Python-Notebooks, MES-Auswertungen, externen Beratungen – und dem Nichtstun.

Deshalb differenzieren wir uns **nicht nur über „KI"**:

- Nutzung durch Produktionsfachleute statt ausschließlich Data Scientists

- verständliche und nachvollziehbare Ergebnisse
- vorhandene Daten statt jahrelanger Standardisierungsprojekte
- kurze Zeit bis zur ersten Erkenntnis
- industrielle Sicherheits- und Betriebskonzepte
- Überführung von Erkenntnissen in konkrete Maßnahmen
